# Оптимизация RAG-системы и контроль качества ответов (Кейс №2)

Проект посвящен глубокой настройке логики ИИ-агента для онлайн-школы. Основной упор сделан на безопасность данных и чистоту пользовательского опыта (UX).

## Проблема

На старте система выдавала ответы с техническим "мусором": транслировала внутренние расчеты (например, 50,000 - 10%), упоминала названия блоков данных и пыталась использовать внешние знания модели для ответа на специфические вопросы о скидках.

## Реализованные решения

* **Внедрение Knowledge Cutoff (Отсечка знаний)**: Установлен жесткий запрет на использование моделью внутренних "знаний" для ответов на финансовые вопросы. Система отвечает строго на базе предоставленных файлов.
* **Настройка Negative Constraints (Негативные ограничения)**: Добавлены инструкции по удалению технического "шума" (JSON-скобок) и ссылок на источники данных.
* **Глубокая настройка UX**: Бот обучен выполнять математические расчеты скрыто, выдавая пользователю только финальную, чистую сумму без процесса вычислений.

## Сравнение результатов

| Характеристика | Базовый RAG (До) | Оптимизированный RAG (После) |
| :--- | :--- | :--- |
| **Формат ответа** | Технический текст со скобками, расчеты | Чистый, лаконичный текст |
| **Риск галлюцинаций** | Высокий (модель додумывает скидки) | Минимальный (строгая отсечка по базе) |
| **Математика** | Демонстрация процесса (50-10%) | Выдача финальной суммы |
