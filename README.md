# Оптимизация RAG-системы и контроль качества ответов

Проект посвящен настройке RAG-системы для AI-ассистента онлайн-школы. Основная цель - снизить количество галлюцинаций, убрать технические артефакты из ответов и сделать взаимодействие с пользователем более понятным.

## Tech Stack

- Coze
- Large Language Models (LLM)
- RAG (Retrieval-Augmented Generation)
- Prompt Engineering
- Knowledge Base
- Guardrails
- AI Agent

## Проблема

В исходной версии RAG-система отвечала не всегда корректно.

Были выявлены следующие проблемы:

- **Использование внутренних знаний модели:** модель использовала собственные знания вместо информации из базы знаний.
- **Отображение внутренних вычислений:** в ответах отображались промежуточные расчеты (например, `50 000 - 10%`).
- **Технический шум:** пользователю показывались технические данные и названия блоков базы знаний.
- **Галлюцинации при расчете стоимости:** при вопросах о стоимости курсов модель могла придумывать скидки, которых не было в документации.

## Что было сделано

- **Knowledge Cutoff:** настроен системный промпт, ограничивающий ответы модели информацией только из базы знаний. Это исключило использование внутренних знаний LLM при ответах на вопросы о стоимости.

- **Negative Constraints:** добавлены ограничения, запрещающие вывод технических данных, внутренних вычислений и служебных элементов в ответах.

- **Оптимизация UX:** изменена логика формирования ответов. Математические вычисления выполняются внутри модели, а пользователь получает только итоговую стоимость без промежуточных расчетов.

## Результат

| Характеристика | До оптимизации | После оптимизации |
| --- | --- | --- |
| **Формат ответа** | Технические данные и промежуточные вычисления | Чистый и понятный ответ |
| **Галлюцинации** | Модель могла использовать внутренние знания | Ответы формируются только на основе базы знаний |
| **Расчет стоимости** | Пользователь видел процесс вычислений | Пользователь получает только итоговую сумму |
